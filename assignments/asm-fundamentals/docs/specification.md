# Учебный проект `asm-fundamentals` — 4 шага к пониманию x86-64

`asm-fundamentals` — последовательный учебный проект из 4 шагов нарастающей сложности, охватывающий регистры, память, управляющие конструкции и системные вызовы x86-64/NASM.
Цель: увидеть каждую операцию
Область применения: подготовка к проекту `fastlog`, отработка базовых паттернов ассемблера, верификация понимания ABI и отладки через `gdb`/`strace`.
Тех. ограничения: только Linux x86-64, NASM синтаксис, системные вызовы `read`/`write`/`exit_group`, отсутствие libc, лимит ~20 часов на весь проект.
Ссылка на спеку: System V AMD64 ABI §3.2.3 (Register Usage) - https://refspecs.linuxfoundation.org/elf/x86_64-abi-0.99.pdf; NASM Manual §3 "The NASM Language" - https://www.nasm.us/doc/nasmdoc3.html

Шаг 1: Регистры и арифметика (2-3 часа)
- Задача: Вычислить `(a * b + c) / d` с фиксированными значениями (`a=12, b=3, c=5, d=2`), вывести результат.
- Концепции: Общие регистры (`RAX`, `RBX`, `RCX`, `RDX`), инструкции `MOV`, `ADD`, `SUB`, `IMUL`, `IDIV`, флаги `RFLAGS`.
- Ключевые команды: `mov rax, 12`, `imul rax, rbx`, `idiv rdx`.
- Проверка: `gdb -batch -ex 'break _start' -ex 'run' -ex 'info registers rax' ./step1` → `rax = 20` (0x14).
- Связь с fastlog: Подготовка к арифметике подсчёта байт/строк.

Шаг 2: Память и переменные (3-4 часа)
- Задача: Загрузить массив из 5 чисел из `.data`, вычислить сумму, сохранить в `.bss`, вывести.
- Концепции: Секции `.data`/`.bss`, директивы `dd`/`resq`, адресация `[base]`, `[base+index*4]`, инструкция `LEA`.
- Ключевые команды: `mov rsi, array`, `add rax, [rsi]`, `lea rcx, [rsi+4]`.
- Проверка: `gdb -ex 'x/5dw &array' -ex 'print $rax' ./step2` → сумма в `rax`.
- Связь с fastlog: Буферизация логов, работа с массивами структур.

Шаг 3: Управление потоком (4-5 часов)
- Задача: Найти максимальное значение в массиве и посчитать его вхождения.
- Концепции: `CMP`, `TEST`, условные переходы (`JE`, `JNE`, `JL`, `JG`), цикл через `dec`/`jnz`, метки.
- Ключевые команды: `cmp rax, [rcx]`, `jg .update_max`, `dec r11`, `jnz .loop`.
- Проверка: `gdb -ex 'break .update_max' -ex 'run' -ex 'print $rax' ./step3`.
- Связь с fastlog: Цикл обработки строк, условный подсчёт по коду ответа.

Шаг 4: Ввод-вывод и интеграция (5-6 часов)
- Задача: Интерактивный калькулятор: прочитать два числа из `stdin`, вывести сумму и произведение.
- Концепции: `syscall` (`read`=0, `write`=1, `exit_group`=231), буфер в `.bss`, конверсия ASCII↔integer, обработка `EOF`.
- Ключевые команды: `syscall`, `cmp byte [rcx], 10`, `sub al, '0'`, `imul rax, 10`.
- Проверка: `echo -e "12\n7" | ./step4` → вывод `Sum: 19`, `Prod: 84`.
- Связь с fastlog: Полный цикл `read`→`process`→`write`, основа Спринта 1.

Итоговая программа: `asm_calc` — CLI-калькулятор, использующий все изученные концепции: регистры, память, циклы, условия, системные вызовы.
