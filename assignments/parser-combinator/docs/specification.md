# Постановка задачи: продолжение lambda-parser

- Реализовать функциональный парсер-комбинатор на базе интерфейса `Parser<A>` с сигнатурой `Optional<Parsed<A>> apply(String, int)`.
- Обеспечить минимальный набор комбинаторов: конкатенация (`plus`), альтернатива (`or`), повтор (`repeat`), опциональность, пропуск (`skipLeft`/`skipRight`).
- Реализовать механизм рекурсивных ссылок через `ProxyParser` для поддержки взаимной рекурсии в eBNF-правилах.
- Добавить валидатор полного разбора (`eof`) и отладочный комбинатор (`debug`) с структурным логированием.
- Собрать парсер арифметических выражений по заданной грамматике: `exp -> sum -> mul -> prime -> number | '(' exp ')'`.
- Обеспечить сравнение объема кода и поддерживаемости с процедурной реализацией (целевой метрикой - сокращение на 30–50%).
