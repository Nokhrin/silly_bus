# Parser Combinator - план работ

Спринт 1: Фундамент (Core & Atomics)
1.  Реализация контрактов: `Parser<A>` (интерфейс), `Parsed<A>` (record)
2.  Исправление атомарных парсеров: `char(char)`, `digit()` (без аргументов, любой), `string(String)`, `whitespace()`
3.  Написание покрывающих тестов для всех атомарных парсеров (граничные условия, смещения, отказы)

Спринт 2: Комбинаторы последовательности и альтернативы
1.  Реализация `plus(right)` (или `then`): последовательное выполнение `Left -> Right`. Управление передачей `offset`
2.  Реализация `or(right)`: выбор `Left`, при отказе попытка `Right`
3.  Введение `map(Function)`: трансформация значения `Parsed` без изменения смещения
4.  Реализация модификаторов `skipLeft` и `skipRight` (отбрасывание части результата)

Спринт 3: Повторы и Опциональность
1.  Реализация `repeat(min, max)`: цикл выполнения парсера
2.  Реализация `zeroOrMore()` (итерация `*` в eBNF) и `oneOrMore()` (итерация `+`)
3.  Реализация `optionally()` (опциональность `?`)
4.  Тестирование комбинаций (например, список чисел, разделенных запятыми)

Спринт 4: Инфраструктура и Рекурсия
1.  Реализация `ProxyParser` (Lazy initialization) для решения проблемы циклических ссылок (взаимная рекурсия правил)
2.  Реализация `eof()`: проверка конца входной строки
3.  Реализация `debug(String name)`: логирование входов/выходов парсера для отладки
4.  Написание теста на рекурсивную грамматику (вложенные скобки)

Спринт 5: Приложение (Калькулятор)
1.  Сборка парсера арифметических выражений (exp -> sum -> mul -> prime)
2.  Интеграционное тестирование валидных и невалидных выражений
3.  Сбор метрик: объем кода, сравнение с процедурным подходом
