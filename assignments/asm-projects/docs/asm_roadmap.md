# Соглашения
## Префиксы Git-коммитов

| Префикс    | Назначение                          | Пример                                      |
|------------|-------------------------------------|---------------------------------------------|
| `feat`     | Новая функциональность              | `feat: реализован factorial(5)`             |
| `fix`      | Исправление ошибки                  | `fix: корректное сохранение RBX`            |
| `refactor` | Рефакторинг без изменения поведения | `refactor: упрощен пролог функции`          |
| `docs`     | Обновление документации             | `docs: добавлен шаг 3.1 в specification.md` |
| `style`    | Форматирование, отступы             | `style: выравнивание инструкций`            |
| `test`     | Добавление тестов                   | `test: проверка factorial(1..5)`            |
| `chore`    | Вспомогательные изменения           | `chore: обновлен .gitignore`                |
| `perf`     | Оптимизация производительности      | `perf: уменьшено число push/pop`            |
| `ci`       | Изменения CI/CD                     | `ci: добавлен Makefile target`              |

## Формат коммита
```
<тип>(<область>): <описание>

[опциональное тело]
```

## Примеры из проекта
```shell
git commit -m "feat: реализован шаг 2.4 (массивы .data/.bss/стека)"
git commit -m "fix: корректное сохранение callee-saved регистров"
git commit -m "docs: обновлена таблица регистров в asm_reference.md"
```
