## Инструкции

### Пересылка данных

| Мнемоника | Формат операндов | Описание                            | Размер результата |
|-----------|------------------|-------------------------------------|-------------------|
| MOV       | reg, reg/mem/imm | Копирование                         | Без изменений     |
| MOVZX     | reg, reg/mem     | Копирование с нулевым расширением   | До размера dest   |
| MOVSX     | reg, reg/mem     | Копирование со знаковым расширением | До размера dest   |
| LEA       | reg, mem         | Загрузка эффективного адреса        | 64 бита           |
| XCHG      | reg, reg         | Обмен значениями                    | Без изменений     |
| PUSH      | reg/mem/imm      | Поместить в стек                    | 64 бита           |
| POP       | reg/mem          | Извлечь из стека                    | 64 бита           |

Варианты размера операнда: byte (8), word (16), dword (32), qword (64). 
Указывается явно: `mov byte [addr], 1`.

### Правило записи

| Запись         | Влияние на RAX       |
|----------------|----------------------|
| `mov al, imm`  | Биты 0-7             |
| `mov ax, imm`  | Биты 0-15            |
| `mov eax, imm` | Биты 0-31, 32-63 = 0 |
| `mov rax, imm` | Биты 0-63            |

32-битная запись (EDI) -> обнуляет старшие 32 бита RDI
64-битная запись (RDI) -> сохраняет все 64 бита

### Арифметические

| Мнемоника | Формат операндов     | Описание                                    | Флаги              |
|-----------|----------------------|---------------------------------------------|--------------------|
| ADD       | reg/mem, reg/mem/imm | Сложение                                    | CF, ZF, SF, OF     |
| SUB       | reg/mem, reg/mem/imm | Вычитание                                   | CF, ZF, SF, OF     |
| INC       | reg/mem              | Инкремент на 1                              | ZF, SF, OF (не CF) |
| DEC       | reg/mem              | Декремент на 1                              | ZF, SF, OF (не CF) |
| NEG       | reg/mem              | Арифметическое отрицание                    | CF, ZF, SF, OF     |
| IMUL      | reg, reg/mem/imm     | Знаковое умножение (результат в RDX:RAX)    | CF, OF             |
| MUL       | reg/mem              | Беззнаковое умножение (результат в RDX:RAX) | CF, OF             |
| IDIV      | reg/mem              | Знаковое деление (результат в RDX:RAX)      | -                  |
| DIV       | reg/mem              | Беззнаковое деление (результат в RDX:RAX)   | -                  |
| CDQ       | -                    | Расширение EAX в EDX:EAX                    | -                  |
| CQO       | -                    | Расширение RAX в RDX:RAX                    | -                  |

Примечание: 
MUL/IMUL с одним операндом используют RAX как аккумулятор, результат в RDX:RAX (128 бит). 
DIV/IDIV делят RDX:RAX на операнд, частное в RAX, остаток в RDX.

### Системные вызовы 

Вызов выполняется по номеру, номер читается из RAX

| Имя        | Задача                      | Номер -> RAX | RDI    | RSI          | RDX             | Изменение RAX   |
|------------|-----------------------------|--------------|--------|--------------|-----------------|-----------------|
| exit       | завершение процесса         | 60           | status | -            | -               | -               |
| exit_group | завершение процесса         | 231          | status | -            | -               | -               |
| read       | чтение RDX элементов из RSI | 0            | fd     | адрес буфера | количество байт | количество байт |
| write      | запись RDX элементов в RSI  | 1            | fd     | адрес буфера | количество байт | количество байт |


`count` - число считанных байт
`exit_group`: Exit code = 1 байт (0-255)
`syscall`:
- перед выполнением записывает RIP -> RCX, RFLAGS -> R11
- после выполнения `SYSRET` записывает RCX -> RIP !!! не обнуляет RCX !!!

##  Файловые дескрипторы - fd

| fd | Имя    | Назначение               | Пример использования       |
|----|--------|--------------------------|----------------------------|
| 0  | stdin  | Стандартный ввод         | `mov rdi, 0` перед `read`  |
| 1  | stdout | Стандартный вывод        | `mov rdi, 1` перед `write` |
| 2  | stderr | Стандартный вывод ошибок | `mov rdi, 2` перед `write` |

Пример:
```nasm
; Чтение из stdin
mov rax, 0          ; sys_read
mov rdi, 0          ; fd = stdin
mov rsi, buf        ; буфер
mov rdx, 10         ; количество байт
syscall

; Запись в stdout
mov rax, 1          ; sys_write
mov rdi, 1          ; fd = stdout
mov rsi, buf        ; буфер
mov rdx, 10         ; количество байт
syscall
```

Примечание:
- Файловые дескрипторы возвращаются syscall `open` (номер 2), `socket` (номер 41)
- Закрытие: `close` (номер 3), аргумент в RDI
- Ошибка: возврат -1 (0xFFFFFFFFFFFFFFFF), код ошибки в errno

## Флаги - eflags

Бит 0: CF (Carry Flag) — перенос при сложении/сдвиге
Бит 1: всегда установлен в 1. Биты 12-14, 15, 17-18, 32-63 зарезервированы процессором.
Бит 2: PF (Parity Flag) — чётность младшего байта результата
Бит 4: AF (Adjust Flag) — вспомогательный перенос (BCD-арифметика)
Бит 6: ZF (Zero Flag) — результат операции равен нулю
Бит 7: SF (Sign Flag) — старший бит результата (знак)
Бит 8: TF (Trap Flag) — пошаговая трассировка
Бит 9: IF (Interrupt Flag) — разрешение аппаратных прерываний
Бит 10: DF (Direction Flag) — направление строковых операций
Бит 11: OF (Overflow Flag) — знаковое переполнение

## Массив

Массив не является примитивом ISA x86-64, ABI или синтаксиса NASM. 
Это паттерн использования памяти, составленный из: 
(1) директив размещения данных, 
(2) режимов адресации, 
(3) дисциплины программиста.

Массив - композиция элементов:
1. Размещение в памяти
2. Доступ по индексу
3. Границы и размер

```asm
; 1. Объявление (NASM §3.2)
section .data
array dd 10, 20, 30, 40, 50   ; 5 элементов, 4 байта каждый

; 2. Получение базового адреса (NASM §3.7)
lea rsi, [array]              ; rsi = &array[0]

; 3. Доступ к элементу (Intel SDM Vol. 1 §3.4)
mov ecx, 2                    ; индекс
mov eax, [rsi + rcx*4]        ; eax = array[2]

; 4. Проверка границ
cmp ecx, 5
jge .out_of_bounds

; 5. Передача в функцию (System V ABI §3.2.3)
mov rdi, rsi                  ; 1-й аргумент = указатель на массив
mov rsi, 5                    ; 2-й аргумент = длина
call process_array
```

## Функции

### Базовый уровень - без call/ret - Метки как функции

| Компонент     | Инструкция    | Назначение             |
|---------------|---------------|------------------------|
| Вход          | `jmp .label`  | Переход к коду функции |
| Аргументы     | RDI, RSI, RDX | Передача значений      |
| Возврат       | RAX           | Результат функции      |
| Возврат в код | `jmp .return` | Переход обратно        |

Пример:
```nasm
section .text
global _start
_start:
    mov rdi, 42          ; аргумент
    jmp .double          ; вызов
    
.double:
    shl rdi, 1           ; тело
    mov rax, rdi         ; результат
    jmp .return          ; возврат
    
.return:
    ; RAX = 84
    mov rdi, rax
    mov rax, 231
    syscall
```

| адрес -> значение                    | `x/dw 0x402010`       | `0x402010 <num1>:	42`                                     |
| Объект по адресу - как код           | `x/5i 0x401018`       | 5 инструкций                                              |
| Объект по адресу - как байты         | `x/10xb 0x401018`     | 10 байт hex                                               |
| Строка (до \0)                       | `x/s 0x401018`        | До первого null-байта                                     |
| Символы (N байт)                     | `x/20cb 0x401018`     | Ровно 20 символов                                         |

### Таблица форматов команды `x` (examine memory)

| Команда           | Count            | Format            | Size                 | Значение                         | Пример вывода                 |
|-------------------|------------------|-------------------|----------------------|----------------------------------|-------------------------------|
| `x/dw 0x402010`   | 1 (по умолчанию) | `d` (decimal)     | `w` (word = 4 байта) | 4 байта как знаковое десятичное  | `0x402010: 42`                |
| `x/5i 0x401018`   | `5` инструкций   | `i` (instruction) | (подразумевается)    | 5 машинных инструкций            | `0x401018: mov rax,1`         |
| `x/10xb 0x401018` | `10` байт        | `x` (hex)         | `b` (byte = 1 байт)  | 10 байт в шестнадцатеричном виде | `0x401018: 0x48 0x83 0xf8...` |
| `x/20cb 0x401018` | `20` байт        | `c` (char)        | `b` (byte = 1 байт)  | 20 байт как ASCII-символы        | `0x401018: 'H' '‰' 'ø'...`    |

### Структура команды `x`

```
x / N F U адрес
    │ │ │
    │ │ └─ Unit size (размер единицы)
    │ └─── Format (формат отображения)
    └───── Count (количество единиц)
```

**Правило:** Общее количество читаемых байт = Count × Unit Size

Пример:
x/20cb 0x401018  →  20 (count) × 1 (b=byte) = 20 байт
x/5xw 0x401018   →  5 (count) × 4 (w=word) = 20 байт
x/3xg 0x401018   →  3 (count) × 8 (g=giant) = 24 байта
