### Флаговый регистр - EFLAGS / RFLAGS
| Мнемоника | Номер бита | Имя флага       | Поведение                                                                       |
|-----------|------------|-----------------|---------------------------------------------------------------------------------|
| CF    | 0          | Carry Flag      | Устанавливается при переносе/заёме из старшего бита (беззнаковое переполнение). |
| PF    | 2          | Parity Flag     | Устанавливается, если младший байт результата содержит чётное число бит 1.      |

## Стек
RBP - адрес первого байта кадра стека
RBP=0 загрузке программы - ядро обнуляет rbp - Согласно System V AMD64 ABI

[RSP] -> адрес вершины стека - адрес для перехода после ret
выделяет ядро при загрузке программы
пример значения: `0x7fffffffd9a0`


The conventional use of %rbp as a frame pointer for the stack frame may be avoided by using
%rsp (the stack pointer) to index into the stack frame

### значение RSP

| Инструкция          | Изменение RSP                  | Размер    | Назначение                    |
|---------------------|--------------------------------|-----------|-------------------------------|
| `call label`        | RSP = RSP - 8                  | 8 байт    | Сохраняет адрес возврата      |
| `ret`               | RSP = RSP + 8                  | 8 байт    | Извлекает адрес возврата      |
| `push reg/mem/imm`  | RSP = RSP - 8                  | 8 байт    | Сохраняет значение в стек     |
| `pop reg/mem`       | RSP = RSP + 8                  | 8 байт    | Извлекает значение из стека   |
| `enter imm16, imm8` | RSP = RSP - imm16              | переменно | Создаёт кадр стека (устарела) |
| `leave`             | RSP = RBP, затем RSP = RSP + 8 | 8 байт    | Уничтожает кадр стека         |


call / ret

call .function
RSP = RSP - 8 (выделение места в стеке)
[RSP] = RIP (сохранение адреса следующей инструкции)
RIP = .function (переход)

.function:
ret
RIP = [RSP] (загрузка адреса возврата)
RSP = RSP + 8 (освобождение места в стеке)





Явно изменяют RSP (программист указывает)

| Инструкция       | Изменение RSP   | Пример                                 |
|------------------|-----------------|----------------------------------------|
| `sub rsp, imm`   | RSP = RSP - imm | `sub rsp, 32` (аллокация локальных)    |
| `add rsp, imm`   | RSP = RSP + imm | `add rsp, 32` (освобождение локальных) |
| `mov rsp, rbp`   | RSP = RBP       | Часть `leave` (восстановление)         |
| `lea rsp, [mem]` | RSP = адрес     | Редко используется                     |

### Передача аргументов в функцию

Push аргументы (в обратном порядке)
push arg3
push arg2
push arg1

call function

Шаг 4: Очистить стек
add rsp, 32        ; 3 arg × 8 + выравнивание

Шаг 5: Получить результат
mov result, rax

### Лимит стека

SAFE_STACK_MARGIN = ожидаемая_глубина × размер_кадра × коэффициент_запаса

где:
- ожидаемая_глубина = максимальное n для рекурсии
- размер_кадра = 8 (адрес возврата) + 8 (saved rbp) + 16 (сохранённые регистры) + 16 (локальные) = 48 байт
- коэффициент_запаса = 10-100 (для безопасности)

### Правило записи

| Запись         | Влияние на RAX       |
|----------------|----------------------|
| `mov al, imm`  | Биты 0-7             |
| `mov ax, imm`  | Биты 0-15            |
| `mov eax, imm` | Биты 0-31, 32-63 = 0 |
| `mov rax, imm` | Биты 0-63            |

32-битная запись (EDI) -> обнуляет старшие 32 бита RDI
64-битная запись (RDI) -> сохраняет все 64 бита

##  Файловые дескрипторы - fd

| fd | Имя    | Назначение               | Пример использования       |
|----|--------|--------------------------|----------------------------|
| 0  | stdin  | Стандартный ввод         | `mov rdi, 0` перед `read`  |
| 1  | stdout | Стандартный вывод        | `mov rdi, 1` перед `write` |
| 2  | stderr | Стандартный вывод ошибок | `mov rdi, 2` перед `write` |

Пример:
```nasm
; Чтение из stdin
mov rax, 0          ; sys_read
mov rdi, 0          ; fd = stdin
mov rsi, buf        ; буфер
mov rdx, 10         ; количество байт
syscall

; Запись в stdout
mov rax, 1          ; sys_write
mov rdi, 1          ; fd = stdout
mov rsi, buf        ; буфер
mov rdx, 10         ; количество байт
syscall
```

Примечание:
- Файловые дескрипторы возвращаются syscall `open` (номер 2), `socket` (номер 41)
- Закрытие: `close` (номер 3), аргумент в RDI
- Ошибка: возврат -1 (0xFFFFFFFFFFFFFFFF), код ошибки в errno

- Адресные переходы/циклы: jmp $, call $+5

## Функции

subprogram == callee

### Метки

| Компонент     | Инструкция    | Назначение             |
|---------------|---------------|------------------------|
| Вход          | `jmp .label`  | Переход к коду функции |
| Аргументы     | RDI, RSI, RDX | Передача значений      |
| Возврат       | RAX           | Результат функции      |
| Возврат в код | `jmp .return` | Переход обратно        |

Пример:
```nasm
section .text
global _start
_start:
    mov rdi, 42          ; аргумент
    jmp .double          ; вызов
    
.double:
    shl rdi, 1           ; тело
    mov rax, rdi         ; результат
    jmp .return          ; возврат
    
.return:
    ; RAX = 84
    mov rdi, rax
    mov rax, 231
    syscall
```

### call / ret

Правило ABI: RSP % 16 == 0 требуется перед инструкцией call

call 
процессор записывает значение rip (=адресу инструкции, следующей за call) в стек

RSP = RSP - 8 (выделение места в стеке)
[RSP] = RIP (сохранение адреса следующей инструкции)
RIP = .function (переход)

ret
RIP = [RSP] (загрузка адреса возврата)
RSP = RSP + 8 (освобождение места в стеке)


### Рекурсия
| Регистр | Назначение   | Сохранение              |
|---------|--------------|-------------------------|
| RDI     | Аргумент (n) | Caller-saved            |
| RAX     | Результат    | Caller-saved            |
| RBX     | Временное n  | Callee-saved (push/pop) |

### Структура команды `x`

```
x / N F U адрес
│ │ │
│ │ └─ Unit size (размер единицы)
│ └─── Format (формат отображения)
└───── Count (количество единиц)
```

Правило: Общее количество читаемых байт = Count × Unit Size

Пример:
x/20cb 0x401018  ->  20 (count) × 1 (b=byte) = 20 байт
x/5xw 0x401018   ->  5 (count) × 4 (w=word) = 20 байт
x/3xg 0x401018   ->  3 (count) × 8 (g=giant) = 24 байта

Печатать 2 значения, каждое в шестнадцатеричном формате, размером 8 байт каждое
(gdb) x/2xg $rsp
0x7fffffffd918:	0x00000000abcdef01	0x0000000012345678

| ASCII->int              | `printf '%d' "'a"`                         | `ord('a')`                    | `97`          |
| int->ASCII              | `printf "\\$(printf '%03o' 97)"`           | `chr(97)`                     | `'a'`         |
| int->bytes (1 байт)     | -                                          | `(97).to_bytes(1, 'big')`     | `b'a'`        |
| bytes->int              | -                                          | `int.from_bytes(b'a', 'big')` | `97`          |

## Проверка секций данных
```shell
readelf -S ./program | grep -E '\.bss|\.data'
# [ 2] .data  PROGBITS  0000000000402000  002000  00000c  0  WA  0   0  1
# [ 3] .bss   NOBITS    000000000040200c  00200c  000010  0  WA  0   0  1
#                                                ↑ выравнивание = 1 байт
```

Расшифровка колонок

| Колонка   | Значение                            | Пример для `.data` |
|-----------|-------------------------------------|--------------------|
| `[Nr]`    | Номер секции                        | `[2]`              |
| `Name`    | Имя секции                          | `.data`            |
| `Type`    | Тип (PROGBITS/NOBITS)               | `PROGBITS`         |
| `Address` | Первый адрес в памяти               | `0x402000`         |
| `Off`     | Смещение в файле                    | `0x2000`           |
| `Size`    | Размер в байтах                     | `0x0c` (12 байт)   |
| `ES`      | Размер элемента (Entry Size)        | `0x00`             |
| `Flg`     | Флаги (W=write, A=alloc, X=execute) | `WA`               |

# Директивы NASM


### Метки

| Тип         | Синтаксис       | Область видимости                  | Пример                                  |
|-------------|-----------------|------------------------------------|-----------------------------------------|
| Глобальная  | Без точки       | Видна везде, экспортируется        | `_start`, `print_hello`                 |
| Локальная   | Одна точка      | Только внутри preceding глобальной | `.loop` внутри `_start` → `_start.loop` |
| Частная     | Две точки + `@` | Не влияет на локальные метки       | `..@foo`                                |
| Специальная | Две точки       | Системные символы                  | `..start`, `..imagebase`                |

Примеры валидных меток

```nasm
_start:         ; глобальная метка (буква в начале)
_main:          ; глобальная с подчёркиванием
.loop:          ; локальная метка (точка в начале)
?temp:          ; метка с вопросом
data$segment:   ; метка с долларом
var@tmp:        ; метка с собакой
label#1:        ; метка с решёткой
```

### Макросы

| Характеристика | `equ`        | `%define`                    |
|----------------|--------------|------------------------------|
| Уровень        | Ассемблер    | Препроцессор                 |
| Перезапись     | Нельзя       | Можно (`%undef` + `%define`) |
| Вычисление     | Compile-time | Текстовая подстановка        |
| GDB видимость  | Не виден     | Не виден                     |

Однострочные
```text
%define ARG1_OFFSET 16
```

Многострочные
```text
%macro  prologue 1 

       push    ebp 
       mov     ebp,esp 
       sub     esp,%1 

%endmacro
```

### Операнды

| Английский термин   | Русский перевод        | Контекст                          |
|---------------------|------------------------|-----------------------------------|
| Memory operand      | Операнд памяти         | Официальный термин в документации |
| Memory access       | Обращение к памяти     | Действие                          |
| Memory operand size | Размер операнда памяти | Количество байт                   |

Пример использования:
```nasm
mov eax, [rbx]    ; [rbx] - операнд памяти (чтение из памяти)
mov [rbx], eax    ; [rbx] - операнд памяти (запись в память)
mov eax, ebx      ; ebx - регистровый операнд (не память)
```

### Указатели размера операндов

| Директива | Байт        | Бит               | Пример                       |
|-----------|-------------|-------------------|------------------------------|
| `byte`    | 1           | 8                 | `mov byte [addr], 1`         |
| `word`    | 2           | 16                | `mov word [addr], 1`         |
| `dword`   | 4           | 32                | `mov dword [addr], 1`        |
| `qword`   | 8           | 64                | `mov qword [addr], 1`        |
| `tword`   | 10          | 80                | `fld tword [addr]` (FPU)     |
| `oword`   | 16          | 128               | `movdqa oword [addr]` (SSE)  |
| `yword`   | 32          | 256               | `vmovdqa yword [addr]` (AVX) |

### Правило указания размера
| Приёмник | Размер обязателен?       | Пример                   |
|----------|--------------------------|--------------------------|
| Регистр  | Нет (размер из регистра) | `mov rdi, 10`            |
| Память   | Да (неизвестен размер)   | `mov dword [rbp-40], 10` |

# Сборка и линковка для Linux x86-64

### Финальная
```shell
nasm -f elf64 file.asm -o file.o
ld -m elf_x86_64 file.o -o file
```

### Отладочная
```shell
nasm -f elf64 -g -F dwarf -Werror -Wlabel -l file.lst file.asm -o file.o
ld -m elf_x86_64 \
   --verbose \
   --print-output-format \
   --trace \
   --cref \
   file.o -o file
```

### Несколько объектных файлов

```shell
nasm -f elf64 src/lib_io.asm -o lib_io.o
nasm -f elf64 src/step_test.asm -o step_test.o
ld lib_io.o step_test.o -o step_test
```
