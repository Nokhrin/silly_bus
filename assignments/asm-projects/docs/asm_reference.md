## Регистры

### Регистры общего назначения

| 64-бит | 32-бит | 16-бит | 8-бит (low) | 8-бит (high) | Назначение по ABI (Linux x86-64)                        |
|--------|--------|--------|-------------|--------------|---------------------------------------------------------|
| RAX    | EAX    | AX     | AL          | AH           | Возврат функции (return value), аккумулятор арифметики  |
| RBX    | EBX    | BX     | BL          | BH           | Callee-saved - сохранить если используешь в функции     |
| RCX    | ECX    | CX     | CL          | CH           | 4-й аргумент функции, счётчик `loop` (устарело)         |
| RDX    | EDX    | DX     | DL          | DH           | 3-й аргумент функции, старшая часть произведения        |
| RSI    | ESI    | SI     | SIL         | -            | 2-й аргумент функции, источник строковых операций       |
| RDI    | EDI    | DI     | DIL         | -            | 1-й аргумент функции, приёмник строковых операций       |
| RSP    | ESP    | SP     | SPL         | -            | Указатель стека - не использовать для данных            |
| RBP    | EBP    | BP     | BPL         | -            | База кадра (callee-saved), можно использовать как общий |
| R8     | R8D    | R8W    | R8B         | -            | 5-й аргумент функции (caller-saved)                     |
| R9     | R9D    | R9W    | R9B         | -            | 6-й аргумент функции (caller-saved)                     |
| R10    | R10D   | R10W   | R10B        | -            | 4-й аргумент syscall (caller-saved, заменяет RCX)       |
| R11    | R11D   | R11W   | R11B        | -            | Clobbered syscall - не сохранять значение после syscall |
| R12    | R12D   | R12W   | R12B        | -            | Callee-saved - для долгосрочных значений между call     |
| R13    | R13D   | R13W   | R13B        | -            | Callee-saved - для долгосрочных значений между call     |
| R14    | R14D   | R14W   | R14B        | -            | Callee-saved - для долгосрочных значений между call     |
| R15    | R15D   | R15W   | R15B        | -            | Callee-saved - для долгосрочных значений между call     |

Запись в 32-битный регистр (EAX) обнуляет старшие 32 бита 64-битного (RAX). 
Запись в 8-битный (AL) затрагивает только младшие 8 бит.

### Выбор регистры для задач

| Задача                | Рекомендуемые регистры       | Почему                              |
|-----------------------|------------------------------|-------------------------------------|
| Аргументы функции     | RDI, RSI, RDX, RCX, R8, R9   | Требование ABI                      |
| Возврат из функции    | RAX                          | Требование ABI                      |
| Временные вычисления  | RAX, RCX, RDX, R10, R11      | Caller-saved, не нужно сохранять    |
| Долгосрочные значения | R12, R13, R14, R15           | Callee-saved, сохранятся после call |
| Счётчик цикла         | RCX (или любой caller-saved) | Исторически, но не обязательно      |
| Адрес массива         | RBX или R12-R15              | Callee-saved, не затрётся в call    |
| Индекс массива        | RCX, RDX, R10, R11           | Caller-saved, достаточно для цикла  |

---

### Caller-saved vs Callee-saved (памятка)

| Тип          | Регистры                                  | Кто сохраняет                      | Когда использовать               |
|--------------|-------------------------------------------|------------------------------------|----------------------------------|
| Caller-saved | RAX, RCX, RDX, RSI, RDI, R8, R9, R10, R11 | Вызывающий (если нужно после call) | Временные значения, аргументы    |
| Callee-saved | RBX, RBP, R12, R13, R14, R15              | Вызываемый (в прологе/эпилоге)     | Долгосрочные значения между call |

### Syscall vs Function Call

| Регистр       | Function Call | Syscall       |
|---------------|---------------|---------------|
| 1-й аргумент  | RDI           | RDI           |
| 2-й аргумент  | RSI           | RSI           |
| 3-й аргумент  | RDX           | RDX           |
| 4-й аргумент  | RCX           | R10 (не RCX!) |
| 5-й аргумент  | R8            | R8            |
| 6-й аргумент  | R9            | R9            |
| Номер/возврат | RAX           | RAX           |
| Clobbered     | -             | R11, RCX      |

### Сегментные регистры

| Мнемоника | Имя          | Назначение по ABI                    |
|-----------|--------------|--------------------------------------|
| CS        | Code Segment | Адрес сегмента с машинными командами |



### Флаговый регистр - rflags

Бит 0: CF (Carry Flag) - перенос при сложении/сдвиге
Бит 1: всегда установлен в 1. Биты 12-14, 15, 17-18, 32-63 зарезервированы процессором.
Бит 2: PF (Parity Flag) - чётность младшего байта результата
Бит 4: AF (Adjust Flag) - вспомогательный перенос (BCD-арифметика)
Бит 6: ZF (Zero Flag) - результат операции равен нулю
Бит 7: SF (Sign Flag) - старший бит результата (знак)
Бит 8: TF (Trap Flag) - пошаговая трассировка
Бит 9: IF (Interrupt Flag) - разрешение аппаратных прерываний
Бит 10: DF (Direction Flag) - направление строковых операций
Бит 11: OF (Overflow Flag) - знаковое переполнение

## Стек

SS -> сегмент, содержащий текущий стек
RBP -> база (адрес первого байта) стека
RSP -> вершина стека
[RSP] -> адрес вершины стека - адрес для перехода после ret

### Выравнивание
обеспечение тождества `rsp % 16 = 0`

Неявно изменяют RSP

| Инструкция          | Изменение RSP                  | Размер    | Назначение                    |
|---------------------|--------------------------------|-----------|-------------------------------|
| `call label`        | RSP = RSP - 8                  | 8 байт    | Сохраняет адрес возврата      |
| `ret`               | RSP = RSP + 8                  | 8 байт    | Извлекает адрес возврата      |
| `push reg/mem/imm`  | RSP = RSP - 8                  | 8 байт    | Сохраняет значение в стек     |
| `pop reg/mem`       | RSP = RSP + 8                  | 8 байт    | Извлекает значение из стека   |
| `enter imm16, imm8` | RSP = RSP - imm16              | переменно | Создаёт кадр стека (устарела) |
| `leave`             | RSP = RBP, затем RSP = RSP + 8 | 8 байт    | Уничтожает кадр стека         |

Явно изменяют RSP (программист указывает)

| Инструкция       | Изменение RSP   | Пример                                 |
|------------------|-----------------|----------------------------------------|
| `sub rsp, imm`   | RSP = RSP - imm | `sub rsp, 32` (аллокация локальных)    |
| `add rsp, imm`   | RSP = RSP + imm | `add rsp, 32` (освобождение локальных) |
| `mov rsp, rbp`   | RSP = RBP       | Часть `leave` (восстановление)         |
| `lea rsp, [mem]` | RSP = адрес     | Редко используется                     |

### Передача аргументов в функцию

Push аргументы (в обратном порядке)
push arg3
push arg2
push arg1

call function

Шаг 4: Очистить стек
add rsp, 32        ; 3 arg × 8 + выравнивание

Шаг 5: Получить результат
mov result, rax

## Инструкции


### Системные вызовы

Вызов выполняется по номеру, номер читается из RAX

| Имя        | Задача                      | Номер -> RAX | RDI    | RSI          | RDX             | Изменение RAX   |
|------------|-----------------------------|--------------|--------|--------------|-----------------|-----------------|
| exit       | завершение процесса         | 60           | status | -            | -               | -               |
| exit_group | завершение процесса         | 231          | status | -            | -               | -               |
| read       | чтение RDX элементов из RSI | 0            | fd     | адрес буфера | количество байт | количество байт |
| write      | запись RDX элементов в RSI  | 1            | fd     | адрес буфера | количество байт | количество байт |


`count` - число считанных байт
`exit_group`: Exit code = 1 байт (0-255)
`syscall`:
- перед выполнением записывает RIP -> RCX, RFLAGS -> R11
- после выполнения `SYSRET` записывает RCX -> RIP !!! не обнуляет RCX !!!

Загрузить значение из памяти по адресу RBX

Загрузить адрес (вычислить выражение, не обращаться к памяти)

Значение регистра RBX

[rbx]
Значение по адресу в RBX

Значение по адресу RBX + RCX*4

указание размера целевого значения
```

```

### Правило записи

| Запись         | Влияние на RAX       |
|----------------|----------------------|
| `mov al, imm`  | Биты 0-7             |
| `mov ax, imm`  | Биты 0-15            |
| `mov eax, imm` | Биты 0-31, 32-63 = 0 |
| `mov rax, imm` | Биты 0-63            |

32-битная запись (EDI) -> обнуляет старшие 32 бита RDI
64-битная запись (RDI) -> сохраняет все 64 бита

| Мнемоника | Формат операндов     | Описание                                    | Флаги              |
|-----------|----------------------|---------------------------------------------|--------------------|

##  Файловые дескрипторы - fd

| fd | Имя    | Назначение               | Пример использования       |
|----|--------|--------------------------|----------------------------|
| 0  | stdin  | Стандартный ввод         | `mov rdi, 0` перед `read`  |
| 1  | stdout | Стандартный вывод        | `mov rdi, 1` перед `write` |
| 2  | stderr | Стандартный вывод ошибок | `mov rdi, 2` перед `write` |

Пример:
```nasm
; Чтение из stdin
mov rax, 0          ; sys_read
mov rdi, 0          ; fd = stdin
mov rsi, buf        ; буфер
mov rdx, 10         ; количество байт
syscall

; Запись в stdout
mov rax, 1          ; sys_write
mov rdi, 1          ; fd = stdout
mov rsi, buf        ; буфер
mov rdx, 10         ; количество байт
syscall
```

Примечание:
- Файловые дескрипторы возвращаются syscall `open` (номер 2), `socket` (номер 41)
- Закрытие: `close` (номер 3), аргумент в RDI
- Ошибка: возврат -1 (0xFFFFFFFFFFFFFFFF), код ошибки в errno



## $ - location counter
адрес текущей позиции ассемблирования
Применение:
- Расчёт длины блока: len equ $ - data_start
- Адресные переходы/циклы: jmp $, call $+5
- Выравнивание/заполнение: times ($-start) % 16 db 0 (или аналоги)


## Константы

message         db      'hello, world'
msglen          equ     $-message

# Директивы NASM


### Метки

| Тип         | Синтаксис       | Область видимости                  | Пример                                  |
|-------------|-----------------|------------------------------------|-----------------------------------------|
| Глобальная  | Без точки       | Видна везде, экспортируется        | `_start`, `print_hello`                 |
| Локальная   | Одна точка      | Только внутри preceding глобальной | `.loop` внутри `_start` → `_start.loop` |
| Частная     | Две точки + `@` | Не влияет на локальные метки       | `..@foo`                                |
| Специальная | Две точки       | Системные символы                  | `..start`, `..imagebase`                |

Примеры валидных меток

```nasm
_start:         ; глобальная метка (буква в начале)
_main:          ; глобальная с подчёркиванием
.loop:          ; локальная метка (точка в начале)
?temp:          ; метка с вопросом
data$segment:   ; метка с долларом
var@tmp:        ; метка с собакой
label#1:        ; метка с решёткой
```

### Операнды

| Английский термин   | Русский перевод        | Контекст                          |
|---------------------|------------------------|-----------------------------------|
| Memory operand      | Операнд памяти         | Официальный термин в документации |
| Memory access       | Обращение к памяти     | Действие                          |
| Memory operand size | Размер операнда памяти | Количество байт                   |

Пример использования:
```nasm

mov eax, ebx      ; ebx - регистровый операнд (не память)
```

## Массив

Индексация
buffer[i] = base_address + (index * element_size)

; Проверка первых 3 элементов
x/3dw $rbx
x/3db $rdi

```
; 1. Объявление (NASM §3.2)
section .data
array dd 10, 20, 30, 40, 50   ; 5 элементов, 4 байта каждый
```

Массив не является примитивом ISA x86-64, ABI или синтаксиса NASM. 
Это паттерн использования памяти, составленный из: 
(1) директив размещения данных, 
(2) режимов адресации, 
(3) дисциплины программиста.

Массив - композиция элементов:
1. Размещение в памяти
2. Доступ по индексу
3. Границы и размер

```asm

; 2. Получение базового адреса (NASM §3.7)
lea rsi, [array]              ; rsi = &array[0]

; 3. Доступ к элементу (Intel SDM Vol. 1 §3.4)
mov ecx, 2                    ; индекс
mov eax, [rsi + rcx*4]        ; eax = array[2]

; 4. Проверка границ
cmp ecx, 5
jge .out_of_bounds

; 5. Передача в функцию (System V ABI §3.2.3)
mov rdi, rsi                  ; 1-й аргумент = указатель на массив
mov rsi, 5                    ; 2-й аргумент = длина
call process_array
```

### Метки как функции

| Компонент     | Инструкция    | Назначение             |
|---------------|---------------|------------------------|
| Вход          | `jmp .label`  | Переход к коду функции |
| Аргументы     | RDI, RSI, RDX | Передача значений      |
| Возврат       | RAX           | Результат функции      |
| Возврат в код | `jmp .return` | Переход обратно        |

Пример:
```nasm
section .text
global _start
_start:
    mov rdi, 42          ; аргумент
    jmp .double          ; вызов
    
.double:
    shl rdi, 1           ; тело
    mov rax, rdi         ; результат
    jmp .return          ; возврат
    
.return:
    ; RAX = 84
    mov rdi, rax
    mov rax, 231
    syscall
```

### call / ret - вызов процедур

call
RSP = RSP - 8 (выделение места в стеке)
[RSP] = RIP (сохранение адреса следующей инструкции)
RIP = .function (переход)

ret
RIP = [RSP] (загрузка адреса возврата)
RSP = RSP + 8 (освобождение места в стеке)

### Структура команды `x`

```
x / N F U адрес
    │ │ │
    │ │ └─ Unit size (размер единицы)
    │ └─── Format (формат отображения)
    └───── Count (количество единиц)
```

Правило: Общее количество читаемых байт = Count × Unit Size

Пример:
x/20cb 0x401018  ->  20 (count) × 1 (b=byte) = 20 байт
x/5xw 0x401018   ->  5 (count) × 4 (w=word) = 20 байт
x/3xg 0x401018   ->  3 (count) × 8 (g=giant) = 24 байта
