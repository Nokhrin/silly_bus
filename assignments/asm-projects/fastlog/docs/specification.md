# Учебный проект `fastlog` - 48 часов

## Цель
Понимание стоимости операций, способов оценки и оптимизации выполнения программы. Парсинг, фильтрация и агрегация логов на уровне системных вызовов.

## Задача
Разработка однозадачной CLI-утилиты обработки текстовых/бинарных логов с использованием стандартных дескрипторов 0, 1, 2.

## Тех. ограничения
- Только Linux x86-64, NASM синтаксис
- Системные вызовы `read`/`write`/`exit_group`
- Отсутствие libc
- Лимит ~48 часов на весь проект

## Этап 1: Базовый ввод/вывод и регистры (16 часов)

### Задачи
- Настройка среды: nasm, ld, gdb, strace
- Цикл чтения: syscall read, обработка EOF
- Подсчёт строк и байт в потоке
- Запись результата в stdout через syscall write

### Побитовые операции для парсинга (практическая интеграция)

| Задача | Операция | Бизнес-контекст |
|--------|----------|-----------------|
| Извлечение уровня лога (биты 7-4) | `shr eax, 4` + `and eax, 0xF` | Фильтрация по severity: ERROR=14, WARN=13, INFO=12 |
| Проверка флага "критичный" (бит 0) | `test eax, 1` + `jnz .critical` | Быстрый отбор критических событий без распаковки |
| Маскирование модуля (биты 15-8) | `and eax, 0xFF00` + `shr eax, 8` | Группировка логов по подсистеме |
| Быстрое умножение на степень двойки | `shl eax, 3` (умножение на 8) | Расчёт смещения в буфере фиксированного размера |
| Проверка выравнивания адреса | `and rsp, ~0xF` | Подготовка буфера для SIMD-операций |

### Пример: парсинг упакованного заголовка лога

```
Формат заголовка (32 бита):
[31:28] severity (4 бита)
[27:24] flags (4 бита)
[23:16] module_id (8 бит)
[15:0]  event_code (16 бит)

Вход: 0xC20A01F4
Выход: severity=12 (INFO), flags=2, module=10, event=500
```

### Псевдокод извлечения полей

```
функция parse_log_header(dword header):
; Извлечение severity (биты 31-28)
mov eax, header
shr eax, 28
; eax = severity (0-15)

    ; Извлечение module_id (биты 23-16)
    mov eax, header
    shr eax, 16
    and eax, 0xFF
    ; eax = module_id (0-255)
    
    ; Извлечение event_code (биты 15-0)
    movzx eax, word [header]
    ; eax = event_code (0-65535)
    
    ВЕРНУТЬ severity, module_id, event_code
```

### Наводящие вопросы
1. Почему `shr` используется для извлечения старших битов, а `and` - для маскирования?
2. В чём разница между `test eax, 1` и `cmp eax, 1` для проверки бита?
3. Почему перед `shr` не нужно обнулять старшие биты регистра?
4. Как проверить, что значение помещается в 4 бита (0-15) без сравнения?
5. Почему `shl eax, 3` быстрее `imul eax, 8` на старых процессорах?

## Этап 2: Память, ABI вызовов и сортировка (16 часов)

### Задачи
- Выделение памяти: .bss, syscall brk для динамических буферов
- Передача параметров в функцию: System V AMD64 ABI
- Реализация qsort или insertion sort для массива 32-битных структур
- Сортировка логов по времени/уровню/модулю

### Интеграция с побитовыми операциями
- Сравнение по битовым маскам: `and reg, mask` перед `cmp`
- Быстрая проверка диапазона: `(x & ~mask) == base`
- Упаковка ключа сортировки: `(severity << 24) | (timestamp & 0xFFFFFF)`

### Проверка
```bash
# Сортировка по убыванию severity
./step2_sort --by-severity < test.log | head -10

# Проверка стабильности сортировки
./step2_sort --stable --by-module < test.log
```

## Этап 3: Оптимизация, SIMD и интеграция с HL (16 часов)

### Задачи
- SIMD парсинг: обработка 16 байт за одну инструкцию (SSE2)
- Векторизация побитовых операций: `pand`, `psrlq`, `pmovmskb`
- JNI/Python FFI: вызов asm-функций из высокоуровневого кода
- Бенчмаркинг: замер времени через `rdtsc` или `clock_gettime`

### Пример: векторизованная фильтрация по маске

```nasm
; Вход: 16 байт заголовков в xmm0
; Маска: 0xF0000000 (извлечение severity)
movdqa xmm1, xmm0          ; копия
psrld xmm1, 28             ; сдвиг на 28 бит для всех 4 dword
pand xmm1, xmm_mask_0xF    ; маскирование
; Результат: 4 значения severity в одном регистре
```

## Итоговая программа: `fastlog`

CLI-утилита для обработки логов, использующая:
- Системные вызовы для ввода/вывода
- Побитовые операции для парсинга упакованных структур
- ABI для модульности кода
- Оптимизацию через SIMD для больших объёмов данных

### Примеры использования
```bash
# Подсчёт строк и байт
./fastlog < access.log

# Фильтрация по уровню (severity >= 12)
./fastlog --min-severity 12 < app.log

# Агрегация по модулю
./fastlog --group-by module < app.log | sort | uniq -c

# Бинарный режим (упакованные заголовки)
./fastlog --binary < events.bin > parsed.txt
```
