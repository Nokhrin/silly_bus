# Учебный проект `asm-string` - 16 часов

## Цель
Понимание модели стека компилятора C при вызове функций: 
размещение локальных переменных, сохранение регистров, передача параметров (включая spill на стек), 
выравнивание и пролог/эпилог. Демонстрация последствий нарушения границ буфера и методов защиты.

## Задача
Реализация набора строковых функций (`strlen`, `memcpy`, `strcpy`, `safe_copy`) 
с явным управлением стековым кадром, воспроизводящим поведение `gcc -O0 -fno-omit-frame-pointer`. 
Анализ структуры памяти при вложенных вызовах и демонстрация переполнения с последующей защитой.

## Тех. ограничения
- Только Linux x86-64 - NASM синтаксис
- System V AMD64 ABI - соблюдение
- Отсутствие libc - реализация с нуля
- Лимит ~16 часов 


Пояснение флагов gcc -O0 -fno-omit-frame-pointer
-O0: Полное отключение оптимизаций. Компилятор не переупорядочивает инструкции, не кеширует переменные в регистрах, не удаляет неиспользуемые вычисления. Код отражает исходную логику 1:1.
-fno-omit-frame-pointer: Запрет на оптимизацию указателя кадра. Современные компиляторы часто используют RSP для адресации локальных переменных, экономя RBP. Данный флаг принудительно генерирует стандартный пролог/эпилог с использованием RBP, что сохраняет неизменную структуру кадра на протяжении выполнения функции. Это упрощает трассировку стека (bt в GDB) и отладку.

## Этап 1: Модель стека C-компилятора и базовые операции (6 часов)

| Параметр         | Значение                                                                                                                                                                                                                                                     |
|------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Задача           | Реализовать `strlen` и `memcpy` с прологом/эпилогом, воспроизводящим структуру кадра C-компилятора                                                                                                                                                           |
| Концепции        | `push rbp` -> `mov rbp, rsp` -> `sub rsp, N`, размещение локальных переменных по `[rbp - offset]`, сохранение callee-saved по `[rbp + offset]`, выравнивание `RSP % 16 == 0` перед `call`, передача первых 6 параметров в регистрах, spill остальных на стек |
| Ключевые команды | `mov rbp, rsp`, `sub rsp, 32`, `mov [rbp-8], rdi`, `mov rax, [rbp+16]`, `leave` (эквивалент `mov rsp, rbp` / `pop rbp`)                                                                                                                                      |
| Проверка         | `gdb -ex 'info frame' -ex 'bt full' ./step1_string` -> совпадение раскладки локальных переменных и сохраненных регистров с выводом `gcc -S -fno-omit-frame-pointer`                                                                                          |
