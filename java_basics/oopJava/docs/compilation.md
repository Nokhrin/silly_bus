# Компиляция
Преобразование исходного кода в код, который может выполнить машина (JVM, процессор)

Этапы компиляции
1. лексический анализ
   - разбиение исходного кода на токены - лексемы - минимальные элементы, сохраняющие смысл команды или данных - значения, операторы, пунктуация
   - удаление пустых строк и комментариев
   - выполняется лексером/сканером
2. [синтаксический анализ / парсинг](parsing.md)
   - проверка последовательности считанных лексем на соответствие правилам языка программирования, для которого выполняется компиляция
   - создание абстрактного синтаксического дерева (parse tree / ast) - структуры данных, напраленного графа, отображающего взаимоотношения считанных языковых конструкций
   - задача - проверить соблюдение синтаксиса языка - отступы, скобки, имена ключевых слов, имена переменных - при нарушении генерируются синтаксические ошибки
3. генерация кода, который может быть выполнен машиной
   - машинного кода, байт-кода виртуальной машины
   - выполняется по данным ast
