# Разработка синтаксиса объявления переменных

## Цель
Спроектировать и реализовать формальную грамматику объявлений и присваиваний

## Задачи

- Разработать EBNF-грамматику лексем и синтаксических правил для объявлений и операторов присваивания
- Реализовать парсинг с помощью ANTLR4 и генерацию AST

### Критерии

- Выразительность (семантическая связь с предметной областью)
- Явное управление изменяемостью (константы|переменные)
- Вывод типа в runtime (компилятор выводит тип из значения)
- Отсутствие визуального шума (необходимое и достаточное множество символов в лексеме)

#### htr

"имя переменной" "=" "число"
"имя переменной" "=" "(" "выражение" ")"
```text
x = 1
x = (1 + 2)
```

[ 2026-06-01 19:21:25 ]
переменная - ссылка для литералов и для выражений 

число == выражение (в вырожденной форме)
"имя переменной" "=" "(" "выражение" ")"
==
"имя переменной" "=" "число" | "(" "выражение" ")"
```text
x = 3
y = (x + 2)
z = x * y => x * (x + 2) => 3 * (3 + 2) => 3 * 5 
```

#### создание
разрешено объявление без явного значения, переменная инициализируется нулем
x
==
x = 0


переменная определяется до использования в выражении
инструкции
```text
x = 3
y = (y + 2)
```
вызовут исключение - так как y не определен до вычисления (y+2)


переопределение возможно
```text
x = 3
x = 4
y = (x + 2) => y = (4 + 2)
```



#### имена переменных

операторы
&, |, ^, ~, *, /, +, -
(И-НЕ, ИЛИ-НЕ достаточно, минимально, но громоздко и невыразительно)


лексемы операторов не могут входить в имена переменных 
из школы: 
a + b = b + a
корни многочлена:
x1, x2, ...

правило: имя буквами в смешанном регистре + номер (опционально)
чувствительность к регистру
[a-zA-Z]+[0-9]*
