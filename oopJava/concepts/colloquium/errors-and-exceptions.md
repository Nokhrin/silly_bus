Отличие java Error от java Exception, в плане влияния на грамматику языка java

семантика исключений
по спецификации java 
Error  и Exception - подтипы Throwable
типы ошибок - подтипы Error
типы исключений - подтипы Exception


erroи exception наследуют throwable
находятся на одном уровне иерархии наследования

исключение может быть выброшено командой throw и перехвачено ближайшей командой catch
оператор throw e , где e имеет тип Е, представляющий ошибку, прерывает поток выполнения и передает управление ближайшему совме иному catch с типом C я являющимся надтипом Е, E <: C

error указывает на системную ошибку, не проверяется в compile, не объявляется в методах через throws

пример - переполнение стека

exception предназначен для предполагаемых, потенциально перехватывемых ошибок

исключения делятся на  unchecked - Errors + Runtime Exceptions, в частности, npe, и 
checked - все остальные



влияние на грамматику языка
метод, который может выбросить checked,
обязан либо обработать в try-catch, либо объявить в throws
unchecked можно выбрасывать без объявления, просто throw ...
это грамматическое правило, проверяемое в compile

void readFile() throws IOException {
    throw new IOException();
}


влияние на систему типов
исключения - объекты, поэтому подчиняются правилам наследования и подтипирования

если Е1 <: Е2, то Е1 можно выбросить там,где ожидается Е2

полиморфизм проявляется при перехвате исключения более общим обработчиком

Exception

компилятор для поддержания монотонности и проверки достижимости требует расположения перехватчиков от частного к общему

подкласс при переопределении может  сужать тип объявленных исключений - принцип подстановки L

checked исключения являются частью контракта метода

синтаксическое требование к потоку выполнения - использовать try catchили throws при работе с checked

для программиста
Error - для системных проблем - память, сбои JVM,
в прикладном коде их не следует предвидеть, они на нижнем уровне абстракции
они произойдут или нет независимо от кода, вероятно, поэтому не проверяются в compile

checked Exceptions - для прикладных аномалий, например, потеря http соединения, переполнение массива,
исключения - часть контракта наряду с параме6. возвращаемым значение , и часть логики системы, подобно условным операторам
