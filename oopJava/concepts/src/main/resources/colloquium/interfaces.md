

интерфейс - абстрактный тип, определяющий контракт - семантические ожидания - в виде множества сигнатур, которые может реализовать любой класс в дереве наследования

класс, который implements - реализует - интерфейс, является подтипом данного интерфейса
C <: A

в java класс может наследовать extends строго один класс, и может реализовывать многие интерфейсы
то есть интерфейсы позволяют множественное наследование поведения - методов
это инструмент обхода ограничения одиночного наследования классов

класс, реализующий интерфейс, становится подтипом этого интерфейса

подтипирования через интерфейс отвечает принципу Лисков - реализатор может использоваться там, где ожидается ссылка на интерфейс

интерфейсы не являются родителями реализаторов, поведение добавляется горизонтально любому узлу дерева наследования, расширяя его поведение
это соответствует подходу композиции

интерфейсы поддерживают строгость типизации - логическую целостность - за счет запрета цикличности расширения
компилятор выполняет поиск вглубину по графу налтипов, если встречается узел, встреченный ранее , генерируется ошибка компиляции

interface A extends B {}
interface B extends A {}
ошибка - синтаксически определение верно за счет forward reference, но в compile при семантическом контроле будет ошибка из-за циклической зависимости типа B от самого себя
