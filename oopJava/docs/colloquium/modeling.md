# достаточного и не достаточного моделирования ООП Класса для отображения явления/феномена реального мира

достаточная модель - обладающая необходимыми свойствами и функциями , решающими поставленную задачу
модель должна быть не исчерпывающей, а минимальной и при этом сохраняющей отношения сущностей


кри ерии достаточной модели
корректность - сохранение инвариантов поставленной задачи - файл недоступен, нельзя скачать

полнота - данных достаточно для решения задачи
известно имя файла и значение состояния доступности

отсутвие избыточности - для скачивания не надо знать uptime сервера

карта метро - не показывает рельеф местности, глубину залегания путей, это избыточно
достаточно - ветки, станции, пересадки 

задача скачать файл

недостаточная модель File(name)
избыточная модель File(name, available, created at) + download + getOwnerName
достаточная : File(name, available) + download

общий критерий модели - соответствие цели


верно ли суждение : недостаточная модель - нерабочая модель?
достаточность - мера эффективности программиста?
может ли избыточность сделать модель достаточной?
