# Полиморфизм  
  
## Аксиоматическое определение | Идея | Что это?  
Свойство системы типов, 
позволяющее осуществить замену значения типа
значением подтипа

  
## Прагматическое определение | Воплощение идеи | Для чего это?  
Методика добавления функционала системы без изменения существующего кода 
взаимодействие старого и нового кода без изменения старого

### Повышение надежности выполнения
(полиморфизм как способ абстрагирования)

[Пример - Хрупкость API RandomAccessFile](../../concepts/src/main/java/org/example/abstraction/FileOperations.java)

### Поддержка выполнения требования | Соблюдения контракта | Работоспособности интерфейса
полиморфизм создает связь классов с помощью системы типов


### Добавление методов
(полиморфизм как способ реализации open/closed principle)
ситуация: Требования к системе изменились - контракт дополнили
профит полиморфизма: уменьшение количества строк кода: +1 тип -> +1 класс, вместо +N точек вызова, которые нужно исправить

#### пример
Задача - выбор сериализатора
для добавления сериализации нового формата создаю **новый класс**, не трогаю существующие *.class  
[Динамический выбор / Полиморфизм](../../concepts/src/main/java/org/example/polymorphism/serializables/SerializableDynamic.java)

#### контрпример
для добавления сериализации нового формата меняю **существующий класс**, необходимо перекомпилировать систему  
[Статический выбор / НеПолиморфизм](../../concepts/src/main/java/org/example/polymorphism/serializables/SerializableStatic.java)


### Однократная верификация значений  
(полиморфизм как способ реализации inheritance)

профит полиморфизма: логика проверок - в корневом классе, добавление методов в интерфейс не потребует повтора логики проверок

ситуация: множество критичных бизнес-требований - баланс должен быть положительным после операции, сумма должна быть неотрицательной
