# Как моделируется предметная область в терминах типов данных и функций


предметная область в программировании 
или домен
это реальность, для которой создается модель, формальное описание
множество сущностей и их отношений

моделирование - построение отобрадения реальности в модель с сохранением структуры объектов и поведения, то есть, создание упрощенной копии реальности, в которой типы кодируют то что существует, функции - то, что можно делать, при этом сохраняя только необходимые для решаемой задачи отношения

в терминах типов и функций
модель - пара T, F
T - типы - классификация сущностей
F - функции - действия, допустимые для сущностей
реальность отображается на модель с сохранением логики отношений (свойство гомоморфности)

в ООП модель = структура (типы) + поведение (интерфейсы)

пример загрузка файла при доступности файла на сервере

упрощение при моделировании - абстракция - исключение не релевантных задаче свойств и отношений

для задачи загрузки файла модели нужны только свойство доступности и функция скачивания
прочие данные и связи игнорируются

тип - множество допустимых значений + операции над ними

тип File - комбинации имени и доступности, для которых работает функция скачать




 существует ли связь между гомоморфизмом и полиморфизмом? существует ли противоречие между моделированием и полиморфным поведением?
