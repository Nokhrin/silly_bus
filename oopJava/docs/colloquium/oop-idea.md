# Идея ООП  
  
ООП <- множество договоренностей о процессе написания кода  
v  
компоненты <- классы: структура и поведение;  
v  
основные принципы <- договоренности о том, какие свойства поддерживают компоненты: наследование, полиморфизм, инкапсуляция, абстракция  
v  
дополнительные принципы (SOLID, DRY, KISS, ...) <- договоренности о взаимодействии компонентов  
v  
языковые конструкции <- средства реализации принципов: переопределение методов; шаблоны: фабрика, одиночка, ...  
v  
реализация <- построение системы в окружении, из компонентов, по принципам
