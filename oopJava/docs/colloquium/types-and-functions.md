# Типы данных в контексте функций

> Как тип данных связан с функциями (функии в широком смысле) и статической типизацией

определим понятие функции как элемента программирования и вычислений

функция - это операция над величинами и / или состояниями

строже
это математическое отображение из множества входных значений | параметров - область определения 
в множество выходных значений | возвращаемых результатов - область допустимых значений

функция обладает свойством детерминированности, то есть каждое вычисление над одними и теми же параметрами вернет один и тот же результат

функция в чистом виде зависит только от области определения
или
для каждого элемента в области определения существует один образ в области допустимых значений

в программировании функции могут быть неоднозначными | не детерминированными
такие функции изменяют состояние системы в виде побочного эффекта - изменяют переменные из нелокальных областей видимости
мошут зависеть от состояния - глобальных переменных, памяти
могут возвращать не детерминированные значения - например, случайные числа
могут зациклиться

различают чистые функции - соответствующие математическому определению
и процедуры - функции в понятиях прикладного программирования

тип данных определяет, какие значения могут быть параметрами функции, какие - результатами
при строгой типизации если функция принимает int , то передача функции значения иного типа вызовет ошибку

тип результата - в java и python - синоним типа функции
если функция возвращает int, то тип функции - int

тип данных - множество значений + допустимые операции

тип данных определяет возможные операции на величинами этого типа
операция - функция, определенная над типом

статическая типизация строго определяет допустимые операции и запрещает иные операции
статическая типизация - система проверки типов в compile time на основе формальных правил языка

то есть говорит "разрешено что определено"
