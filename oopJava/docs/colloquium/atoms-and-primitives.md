# Атомарные и примитивные типы

> Атомарные (в смысле первичные типы из которых остальные составляются) типы / Примитивные типы

теория
атомарный тип - понятие системы типов
логически неделимое значение, то есть значение, которое не имеет внутренней структуры, определенной через другие типы
не может быть декомпозирован

атомарность типа - отсутствие структуры в грамматике типа


аналогия
буквы алфавита - атомарный тип в системе типов языка, для которого определен алфавит
буквы на пишущей машинке - примитивные типы - реализация атомарного типа - из которых можно набрать слово
слово будет составням типом

в понятиях кс-грамматики , определяющей правила языка, атомарные типы - это терминалы, объекты из которых конструируются правила кс-грамматики



реализация в java
непосредственное хранение - расположение значения в стеке, не по ссылке
примитивный тип в java - реализация атомарного типа системы типов java
значения примитивных типов хранятся непосредственно
java имеет 8 примитивных типов : boolean, byte, short, int, long, double, float, double, char
значения этих типов не являются объектами, хранятся в памяти, не имеют полей и методов
не наследуются от object, не могут быть null, имеют фиксированный размер, поддерживают неявное расширение - преобразование от более узкого к более широкому
int->long

процесс
при вызове метода создается фрейм стека - локальная область видимости функции
примитивы и ссылки функции записываются во фрейм стека 

при создании объекта выделяется область памяти в куче, в эту область записывается заголовок и поля обьекта
статические поля - с целью предоставления доступа - хранятся в иной области памяти - области метаданных
