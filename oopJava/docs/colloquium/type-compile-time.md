фазы жизненного цикла программы java

 написание классов .java - программист, генератор кода

compile time - работа javac по созданию .class из .java - компилятор

runtime
load - первый этап runtime - загрузка .class в JVM, связывание - размещение структур данных класса в method area ,- в том числе vtable, выполнение статических инициализаторрв - JVM

выполнение байт кода - jvm

compile time - фаза жизненного цикла программы, предшествующая выполнению, задача которой -создание исполняемого кода - байт кода JVM в случае java

инициируется вызовом компилятора
явно javac Main.java
или неявно - Maven mvn compile, IDEA с помощью run конфигурации

фаза включает этапы
лексинг - представление ввода как последовательности единиц грамматики - ключевые слова, идентификаторы - без семантики, по ас грамматике
синтаксический анализ исходного кода - парсинг - построение ast по гоамматике

семантический анализ исходного кода - проверка типов, совместимости типов значений

принятие решения - может ли метод быть переопределен - по модификаторам -
если не static, не private, не final, не конструкттр -  метод виртуальный и требует динамического связывания

генерация байт кода
среди прочего компилятор записывает инструкцию invokevirtual для виртуального метода


Где в compile time хранится информация о типе
в compile существует тип переменной

компилятор хранит данные в памяти процесса javac
формат данных - дерево ast, таблицы символов
в таблицах полная информация об объектах
в узлах дерева - имена объектов и ссылки на соответствующие таблицы

информация о типах хранится
 в атрибутах узлов - какой тип результата вычисления узла, если считать код в узле выражением?
 
   и в таблицах символов - где символ - множество атрибутов объявленной сущности - переменной, метода, класса. Тип данных - один из атрибутов символа
