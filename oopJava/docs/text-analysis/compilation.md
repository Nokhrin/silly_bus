# Компиляция
Компиляция - процесс преобразования текста на формальном языке - например, языке программирования -
в машинный код или байт-код, который может выполнить машина
Машинный код может быть обработан физическим процессором, байт-код - виртуальным процессором, например, JVM в Java, V8 в JS

Компиляция - отображение строки символов в машинный код: Σ* -> code
где Σ - алфавит языка
где Σ* - все строки, которые можно составить из символов алфавита языка - не обязательно валидные, любые сочетания

## Задача компиляции 
Проверка соответствия текста [грамматике](grammar.md) языка, на котором написан текст

## Этапы компиляции
> неудача прохождения одного из этапов ведет к неудаче компиляции

1. лексический анализ
   - разбиение исходного кода на токены - лексемы - минимальные элементы, сохраняющие смысл команды или данных - значения, операторы, пунктуация
   - удаление пустых строк и комментариев
   - выполняется лексером/сканером
2. [синтаксический анализ / парсинг](parsing.md)
   - проверка последовательности считанных лексем на соответствие синтаксическим правилам языка программирования, для которого выполняется компиляция
   - создание абстрактного синтаксического дерева (parse tree / ast) - структуры данных, напраленного графа, отображающего взаимоотношения считанных языковых конструкций
   - задача - проверить соблюдение синтаксиса языка - отступы, скобки, имена ключевых слов, имена переменных - при нарушении генерируются синтаксические ошибки
3. генерация кода
   - генератор получает на вход ast
   - генератор преобразует ast в машинный код или байт-код вм
