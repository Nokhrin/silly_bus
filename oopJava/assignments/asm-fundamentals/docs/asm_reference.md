# Регистры и мнемоники x86-64 для asm-fundamentals**

**Примечание:** Справочник содержит избыточные варианты (MUL/IMUL, DIV/IDIV, JL/JB, etc.). Выбор конкретной инструкции и размера регистра принимается программистом на основе задачи. Ошибки выбора (например, DIV без обнуления RDX, MOVZX вместо MOVSX для знаковых данных) проявляются при выполнении.

Тех. ограничения: только Linux x86-64 ABI, синтаксис NASM, без SIMD/расширений.
Ссылка на спеку: Intel SDM Vol. 1 §3.4 "General-Purpose Registers" - https://www.intel.com/content/www/us/en/developer/articles/technical/intel-sdm.html; NASM Manual §3.1 "Basic Syntax" - https://www.nasm.us/doc/nasmdoc3.html

### 1. Регистры общего назначения (64-бит)

| 64-бит | 32-бит | 16-бит | 8-бит (low) | 8-бит (high) | Назначение по ABI                                        |
|--------|--------|--------|-------------|--------------|----------------------------------------------------------|
| RAX    | EAX    | AX     | AL          | AH           | Аккумулятор, возврат syscall                             |
| RBX    | EBX    | BX     | BL          | BH           | Сохраняемый (callee-saved)                               |
| RCX    | ECX    | CX     | CL          | CH           | Счётчик цикла (неявно декрементируется инструкцией LOOP) |
| RDX    | EDX    | DX     | DL          | DH           | Деление, 3-й аргумент syscall                            |
| RSI    | ESI    | SI     | SIL         | -            | 2-й аргумент syscall                                     |
| RDI    | EDI    | DI     | DIL         | -            | 1-й аргумент syscall                                     |
| RBP    | EBP    | BP     | BPL         | -            | Базовый указатель стека                                  |
| RSP    | ESP    | SP     | SPL         | -            | Указатель стека                                          |
| R8     | R8D    | R8W    | R8B         | -            | 4-й аргумент syscall                                     |
| R9     | R9D    | R9W    | R9B         | -            | 5-й аргумент syscall                                     |
| R10    | R10D   | R10W   | R10B        | -            | 6-й аргумент syscall                                     |
| R11    | R11D   | R11W   | R11B        | -            | Временный (clobbered syscall)                            |
| R12    | R12D   | R12W   | R12B        | -            | Сохраняемый (callee-saved)                               |
| R13    | R13D   | R13W   | R13B        | -            | Сохраняемый (callee-saved)                               |
| R14    | R14D   | R14W   | R14B        | -            | Сохраняемый (callee-saved)                               |
| R15    | R15D   | R15W   | R15B        | -            | Сохраняемый (callee-saved)                               |

**Примечание:** Запись в 32-битный регистр (EAX) обнуляет старшие 32 бита 64-битного (RAX). Запись в 8-битный (AL) затрагивает только младшие 8 бит.

### 2. Инструкции пересылки данных

| Мнемоника | Формат операндов | Описание | Размер результата |
|-----------|------------------|----------|-------------------|
| MOV | reg, reg/mem/imm | Копирование | Без изменений |
| MOVZX | reg, reg/mem | Копирование с нулевым расширением | До размера dest |
| MOVSX | reg, reg/mem | Копирование со знаковым расширением | До размера dest |
| LEA | reg, mem | Загрузка эффективного адреса | 64 бита |
| XCHG | reg, reg | Обмен значениями | Без изменений |
| PUSH | reg/mem/imm | Поместить в стек | 64 бита |
| POP | reg/mem | Извлечь из стека | 64 бита |

**Варианты размера операнда:** byte (8), word (16), dword (32), qword (64). Указывается явно: `mov byte [addr], 1`.

### 3. Арифметические инструкции

| Мнемоника | Формат операндов     | Описание                                    | Флаги              |
|-----------|----------------------|---------------------------------------------|--------------------|
| ADD       | reg/mem, reg/mem/imm | Сложение                                    | CF, ZF, SF, OF     |
| SUB       | reg/mem, reg/mem/imm | Вычитание                                   | CF, ZF, SF, OF     |
| INC       | reg/mem              | Инкремент на 1                              | ZF, SF, OF (не CF) |
| DEC       | reg/mem              | Декремент на 1                              | ZF, SF, OF (не CF) |
| NEG       | reg/mem              | Арифметическое отрицание                    | CF, ZF, SF, OF     |
| IMUL      | reg, reg/mem/imm     | Знаковое умножение (результат в RDX:RAX)    | CF, OF             |
| MUL       | reg/mem              | Беззнаковое умножение (результат в RDX:RAX) | CF, OF             |
| IDIV      | reg/mem              | Знаковое деление (результат в RDX:RAX)      | -                  |
| DIV       | reg/mem              | Беззнаковое деление (результат в RDX:RAX)   | -                  |
| CDQ       | -                    | Расширение EAX в EDX:EAX                    | -                  |
| CQO       | -                    | Расширение RAX в RDX:RAX                    | -                  |

**Примечание:** 
MUL/IMUL с одним операндом используют RAX как аккумулятор, результат в RDX:RAX (128 бит). 
DIV/IDIV делят RDX:RAX на операнд, частное в RAX, остаток в RDX.

### 4. Инструкции сравнения и управления потоком

| Мнемоника | Формат операндов | Описание | Флаги |
|-----------|------------------|----------|-------|
| CMP | reg/mem, reg/mem/imm | Сравнение (вычитание без записи) | CF, ZF, SF, OF |
| TEST | reg, reg/mem/imm | Логическое И без записи | CF, ZF, SF, OF |
| JMP | label | Безусловный переход | - |
| JE/JZ | label | Переход если равно/ноль | ZF=1 |
| JNE/JNZ | label | Переход если не равно/не ноль | ZF=0 |
| JL/JNGE | label | Переход если меньше (знаковое) | SF≠OF |
| JLE/JNG | label | Переход если меньше или равно | ZF=1 ∨ SF≠OF |
| JG/JNLE | label | Переход если больше (знаковое) | ZF=0 ∧ SF=OF |
| JGE/JNL | label | Переход если больше или равно | SF=OF |
| JB/JNAE | label | Переход если ниже (беззнаковое) | CF=1 |
| JBE/JNA | label | Переход если ниже или равно | CF=1 ∨ ZF=1 |
| JA/JNBE | label | Переход если выше (беззнаковое) | CF=0 ∧ ZF=0 |
| JAE/JNB | label | Переход если выше или равно | CF=0 |
| LOOP | label | Цикл по RCX (RCX=RCX-1, если ≠0 переход) | - |

### 6. Системные вызовы (Linux x86-64)

| Действие                      | Номер | Мнемоника  | RDI    | RSI | RDX   | R10   | R8 | R9     | Возврат (RAX)              |
|-------------------------------|-------|------------|--------|-----|-------|-------|----|--------|----------------------------|
|                               | 0     | read       | fd     | buf | count | -     | -  | -      | байт (>0), 0 (EOF), -errno |
|                               | 1     | write      | fd     | buf | count | -     | -  | -      | байт (>0), -errno          |
|                               | 60    | exit       | status | -   | -     | -     | -  | -      | не возвращает              |
| завершение потоков в процессе | 231   | exit_group | status | -   | -     | -     | -  | -      | не возвращает              |

**Инструкция:** `syscall` (opcode 0F 05). Перезаписывает RCX (saved RIP) и R11 (saved RFLAGS).

### 7. Директивы секций NASM

| Директива | Описание                    | Выравнивание | Инициализация          |
|-----------|-----------------------------|--------------|------------------------|
| .text     | Код программы               | 16 байт      | -                      |
| .data     | Инициализированные данные   | 1 байт       | Явная (db, dw, dd, dq) |
| .bss      | Неинициализированные данные | 16 байт      | Нули (ядро)            |
| .rodata   | Только для чтения           | 16 байт      | Явная                  |

#### Директивы данных - секция .data

| Директива | Описание        | Выравнивание | Мнемоника |
|-----------|-----------------|--------------|-----------|
| db        | Байт            | 1 байт       | resb      |
| dw        | Слово           | 2 байт       | resw      |
| dd        | Двойное слово   | 4 байт       | resd      |
| dq        | Четверное слово | 8 байт       | resq      |

### 8. Флаги RFLAGS

| Флаг | Бит | Название | Устанавливается |
|------|-----|----------|-----------------|
| CF | 0 | Carry Flag | Перенос/заём |
| ZF | 6 | Zero Flag | Результат = 0 |
| SF | 7 | Sign Flag | Результат < 0 (знаковый) |
| OF | 11 | Overflow Flag | Знаковое переполнение |
| IF | 9 | Interrupt Flag | Разрешение прерываний |

### 9. Массивы

Массив не является примитивом ISA x86-64, ABI или синтаксиса NASM. 
Это паттерн использования памяти, составленный из: 
(1) директив размещения данных, 
(2) режимов адресации, 
(3) дисциплины программиста.

Массив - композиция элементов:
1. Размещение в памяти
2. Доступ по индексу
3. Границы и размер

```asm
; 1. Объявление (NASM §3.2)
section .data
array dd 10, 20, 30, 40, 50   ; 5 элементов, 4 байта каждый

; 2. Получение базового адреса (NASM §3.7)
lea rsi, [array]              ; rsi = &array[0]

; 3. Доступ к элементу (Intel SDM Vol. 1 §3.4)
mov ecx, 2                    ; индекс
mov eax, [rsi + rcx*4]        ; eax = array[2]

; 4. Проверка границ
cmp ecx, 5
jge .out_of_bounds

; 5. Передача в функцию (System V ABI §3.2.3)
mov rdi, rsi                  ; 1-й аргумент = указатель на массив
mov rsi, 5                    ; 2-й аргумент = длина
call process_array
```

# Отладка кода

### 0. Собрать с отладочной информацией
```shell
nasm -f elf64 -g src/step1_task.asm -o step1_task.o
ld -no-pie -m elf_x86_64 step1_task.o -o step1_task
```

### 1. Запустить в GDB с остановкой на входе
```shell
gdb -q ./step1_task
break _start
run
```

### 2. Включить автоматический дизассемблер
```shell
display/i $pc
```

### 3. Шагать
```shell
stepi
```

#### 3.1. Проверять значения в регистрах
```shell
display/x $rax
```

#### 3.2. Проверять флаги
```shell
info registers eflags
```

### 5. Поставить точку останова перед syscall


#### 5.1. Узнать адрес для брейкпоинта перед mov rax, 231
```shell
disassemble _start,+100
# 0x0000000000401029 <_start+41>:      mov    $0xe7,%eax - этот адрес
# 0x000000000040102e <_start+46>:      syscall
```

```shell
break *0x401029
continue
print/d $rdi
```

### 6. Проверить системные вызовы
```shell
strace -e exit_group ./step1_task
```

| Команда           | Назначение               | Пример вывода                   |
|-------------------|--------------------------|---------------------------------|
| `info registers`  | Все регистры             | `rax 0x1 1`                     |
| `print/x $rax`    | Значение в 16-ричном     | `$1 = 0x1`                      |
| `print/d $rax`    | Значение в 10-ричном     | `$2 = 1`                        |
| `print/t $rax`    | Бинарное представление   | `$3 = 00000001`                 |
| `x/4xb &var`      | 4 байта памяти по адресу | `0x402000: 0x2a 0x00 0x00 0x00` |

| `stepi`           | Одна инструкция          | —                               |
| `break *0x401010` | Брейкпоинт по адресу     | `Breakpoint 2 at 0x401010`      |
