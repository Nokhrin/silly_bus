# Структура приложения

# Слои

## Парсер команд
задача: создание команд // сырые данные -> представление данных
принимает строку, возвращает список команд с аргументами

## Контроллер команд
задача: создание операций - // представление данных -> бизнес-логика
принимает команду и аргументы, возвращает объект операции

## Контроллер операций
задача: выполнение операций + сохранение состояния системы
принимает объект операции, выполняет операцию, возвращает результат операции, взаимодействует с хранилищем операций

# Хранилище операций
является интерфейсом, позволяет подменять реализацию: память, файл, бд
