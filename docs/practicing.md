## идея проекта  
существует проект, содержащий структурированную документацию  
атом документации - текстовый файл  
кодировка файла: utf8  

содержимое файла: текст в формате markdown, mermaid, latex, plantuml, ebnf  

файл может содержать ссылки на другой файл  
другим файлом может быть любая директория:  
- не только из текущего проекта  
- удаленный файл на веб-сервере  
- удаленный файл на ftp-сервере  
- файл в локальной фс  
- цитаты текста (примеры исходного кода)  


## решаемая проблема  
валидировать связи:  
- проверять что удаленные файлы существуют / ссылки рабочие  
- проверять, что цитируемый текст соответствует источнику  

## возможное решение  

система принимает на вход корневую директорию  
рекурсивно обнаруживает файлы заданных расширений  
извлекает строки, соответствующие шаблону ссылки  
выполняет проверку существования целевого ресурса  
формирует отчет о статусе каждой связи  

### проверять, что цитируемый текст соответствует источнику  
- при цитировании требовать указание пути к файлу, из которого цитата  
- проверять что цитата содержится в файле  
- не менять файлы / сохранять идемпотентность
